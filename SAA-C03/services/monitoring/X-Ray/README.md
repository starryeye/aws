## X-Ray

- 분산 시스템에서 요청 흐름을 추적하고 성능 문제를 분석하는 서비스

---

## 핵심 개념

- 하나의 요청이 여러 서비스(Lambda, EC2, DB 등)를 거치는 흐름을 추적
- 어디서 지연이 발생했는지 확인 가능

---

## 특징

    - 분산 트레이싱 (Distributed Tracing)
    - 서비스 간 호출 흐름 시각화
    - 지연 시간 분석
    - 오류 원인 파악

---

## 주요 개념

---

### 1️⃣ Trace

- 하나의 요청 전체 흐름

---

### 2️⃣ Segment

- 서비스 단위 작업

예:

- API Gateway
- Lambda
- EC2

---

### 3️⃣ Subsegment

- 서비스 내부 작업

예:

- DB 호출
- 외부 API 호출

---

## 동작 흐름

    Client 요청
        ↓
    API Gateway
        ↓
    Lambda
        ↓
    DynamoDB

👉 이 전체 흐름이 하나의 Trace

---

## 특징 정리

- 요청 흐름 전체 추적
- 병목 구간 식별
- 서비스 간 관계 분석

---

## 주요 기능

### 1️⃣ Service Map

- 서비스 간 연결 관계 시각화

---

### 2️⃣ Latency 분석

- 어디서 시간이 많이 걸리는지 확인

---

### 3️⃣ Error 분석

- 오류 발생 위치 확인

---

## 사용 케이스

- 마이크로서비스 디버깅
- 성능 병목 분석
- 요청 흐름 추적

---

## CloudWatch vs X-Ray

| 구분 | CloudWatch | X-Ray |
|---|---|---|
관점 | 시스템 상태 | 요청 흐름 |
단위 | Metric | Trace |
목적 | 모니터링 | 디버깅 |

---

## 한 줄 정리

- X-Ray = "요청이 어디를 거쳤고 어디서 느린지 추적하는 서비스"
