## Managed Service for Apache Flink

- AWS에서 제공하는 완전 관리형 스트리밍 데이터 처리/분석 서비스
    - Apache Flink 기반

---

## 핵심 개념

- 실시간으로 들어오는 데이터를 "즉시 계산/분석/변환"하는 엔진

---

## 역할 (매우 중요)

- 데이터 전송 ❌
- 데이터 수집/저장 ❌
- 데이터 처리/분석 ⭕

---

## 위치 (아키텍처)

    Data Streams / Kafka
        ↓
    Flink (실시간 처리)
        ↓
    S3 / Redshift / OpenSearch

---

## 특징

    - 완전 관리형 (Serverless)
    - 실시간 스트리밍 처리
    - 상태 기반 처리 (Stateful)
    - 자동 확장

---

## 주요 기능

---

### 1️⃣ 실시간 처리

- 데이터가 들어오는 즉시 처리

---

### 2️⃣ Stateful Processing

- 이전 상태를 기억하고 처리 가능

예:

- 사용자 행동 분석
- 세션 기반 처리

---

### 3️⃣ Window 연산

- 일정 시간 단위 집계

예:

- 1분간 이벤트 수

---

### 4️⃣ 복잡한 이벤트 처리

- 패턴 기반 분석

---

## As-Is → To-Be

### ❌ As-Is (Streams만 사용)

    데이터 → Streams → 저장

- 데이터 "흐름"만 있음
- 계산 불가

---

### ✅ To-Be (Flink 추가)

    데이터 → Streams → Flink → 결과 저장

- 실시간 계산 가능
- 분석 / 집계 가능

---

## Streams vs Firehose vs Flink

| 구분   | Data Streams | Firehose | Flink |
|------|--------------|----------|-------|
 역할   | 수집           | 전송       | 처리/분석 |
 실시간성 | ⭕            | 준실시간     | ⭕     |
 처리   | ❌            | ❌        | ⭕     |
 유연성  | 높음           | 낮음       | 매우 높음 |

---
