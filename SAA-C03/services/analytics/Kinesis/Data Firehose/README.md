## Kinesis Data Firehose

- 데이터를 실시간으로 수집하여 S3, Redshift 등으로 자동 전송하는 서비스

---

## 핵심 개념

- 데이터 수집 → 변환 → 저장을 자동으로 처리
- 스트리밍 데이터를 "쌓아주는" 서비스

---

## 특징

    - 완전 관리형 (Serverless)
    - 자동 확장
    - 실시간 데이터 전달
    - 데이터 변환 가능

---

## 동작 흐름

    데이터 생성 (앱 / 로그)
        ↓
    Kinesis Data Firehose
        ↓
    (옵션) Lambda 변환
        ↓
    S3 / Redshift / OpenSearch

---

## 주요 기능

---

### 1️⃣ 데이터 수집

- 애플리케이션 로그
- IoT 데이터
- 이벤트 스트림

---

### 2️⃣ 데이터 변환

- Lambda를 통한 데이터 가공

---

### 3️⃣ 데이터 전달

- S3
- Redshift
- OpenSearch

---

## 특징 정리

- 데이터 자동 적재
- 버퍼링 후 전달 (batch 방식)
- 운영 부담 없음

---

## 사용 케이스

- 로그 수집
- 실시간 데이터 적재
- 데이터 레이크 구축

---

## Firehose vs Kinesis Data Streams

| 구분 | Firehose | Data Streams |
|---|---|---|
관리 | 자동 | 직접 관리 |
처리 | 자동 전달 | 직접 소비 |
실시간성 | 준실시간 | 실시간 |
유연성 | 낮음 | 높음 |

---

## 한 줄 정리

- Firehose = "데이터를 모아서 자동으로 저장소로 보내주는 서비스"
