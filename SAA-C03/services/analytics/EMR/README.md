## EMR (Elastic MapReduce)

- AWS에서 제공하는 빅데이터 처리(compute) 서비스
    - Hadoop / Spark 기반 분산 처리

---

## 핵심 개념

- 여러 EC2를 묶어서 하나의 클러스터로 구성
- 대용량 데이터를 병렬 처리

---

## 특징

    - 완전 관리형 빅데이터 플랫폼
    - 자동 확장 (Auto Scaling)
    - EC2 기반 클러스터
    - 다양한 오픈소스 지원

---

## 지원 프레임워크

- Hadoop
- Spark (가장 많이 사용)
  - 실시간 분석에 적합하지는 않음
- Hive
- HBase
- Presto

---

## 구성 요소

### 1️⃣ Cluster

- 여러 EC2 인스턴스로 구성된 집합

---

### 2️⃣ Master Node

- 클러스터 관리

---

### 3️⃣ Core Node

- 데이터 저장 + 처리

---

### 4️⃣ Task Node

- 처리만 수행 (옵션)

---

## 동작 방식

    S3 / HDFS 데이터
        ↓
    EMR Cluster (분산 처리)
        ↓
    결과 저장 (S3 등)

---

## 특징 정리

- 대용량 데이터 처리
- 분산 처리
- 병렬 처리

---

## 저장소

- S3 (주로 사용)
- HDFS (클러스터 내부)

---

## 사용 케이스

- 로그 분석
- ETL 작업
- 데이터 처리 파이프라인
- 머신러닝 전처리

---

## EMR vs 다른 서비스

| 구분 | EMR | Athena | Redshift |
|---|---|---|---|
성격 | 데이터 처리 | 쿼리 서비스 | 데이터 웨어하우스 |
방식 | 클러스터 기반 | 서버리스 | 저장 + 분석 |
유연성 | 높음 | 낮음 | 중간 |

---

## 한 줄 정리

- EMR = "EC2 클러스터 기반으로 빅데이터를 분산 처리하는 서비스"
