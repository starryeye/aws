## Storage Gateway
- 온프레미스에서 AWS 스토리지를 로컬처럼 사용할 수 있게 해주는 서비스
- 특징
  - 데이터를 옮기는 것이 목적이 아닌, 연결해서 쓰는것이 목적이다.
  - 통로 역할을 할 뿐, 자체적인 저장 기능을 가지고 있지 않음
- 유형
  - file gateway
    - 온프레미스 데이터를 '공유 폴더' 형태로 연동하여 S3 에 저장해주는 gateway
    - NFS, SMB 프로토콜을 사용하여 "파일 자체를" S3 에 업로드하는 방식
  - volume gateway
    - 온프레미스 데이터를 '블록 스토리지(디스크)' 형태로 연동하여 S3 에 저장해주는 gateway
    - 파일이 아닌 "블록 스토리지(EBS) 스냅샷"을 S3 에 업로드하는 방식
      - 파일로 업로드하지 않아 S3 에서 사용 불가
  - tape gateway
    - 온프레미스 의 "물리적 테이프"에 저장된 데이터를 S3 에 저장해주는 게이트웨이
    - 백업 및 아카이브 용도
