## Amazon Macie

- AWS에서 제공하는 **데이터 보안 및 민감 정보 탐지 서비스**
- "S3" 에 저장된 데이터를 분석하여 **민감 정보(PII) 및 데이터 노출 위험 탐지**

---

### 🔍 주요 기능

#### 1. 민감 데이터 탐지 (Sensitive Data Discovery)
- S3 객체를 스캔하여 민감 정보 식별
- 탐지 대상:
    - 개인정보 (PII)
        - 이름, 이메일, 전화번호
        - 주민등록번호, 여권번호 등
    - 금융 정보
        - 신용카드 번호
- **머신러닝 + 패턴 매칭** 기반 분석

---

#### 2. S3 데이터 가시성 (Visibility)
- 버킷 및 객체에 대한 보안 상태 분석
- 제공 정보:
    - 공개 여부 (Public Access)
    - 암호화 여부
    - 데이터 유형 분류

---

#### 3. 이상 접근 탐지
- S3 데이터 접근 패턴 분석
- 비정상적인 접근 활동 탐지

---

### 📊 결과 (Findings)
- 민감 데이터 또는 보안 이슈 발견 시 **Finding 생성**
- 연동:
    - EventBridge → 자동 대응
    - Security Hub → 통합 보안 관리

---

### ⚙️ 특징
- 완전 관리형 서비스 (서버 구성 불필요)
- 자동 스캔 및 주기적 분석
- 데이터 분류 및 보안 상태 시각화 제공

---

### 🔗 연관 서비스
- S3 (분석 대상)
- IAM (접근 제어)
- Security Hub (보안 통합)
- EventBridge / Lambda (자동 대응)

---

### ⚠️ 한계
- **S3 전용 서비스**
    - EC2, RDS 데이터 직접 분석 불가
- **탐지 서비스**
    - 자동 차단 기능 없음

---

### 🧠 핵심 포인트
- Macie = **S3 데이터 보안 + 민감 정보 탐지**
- PII / 금융 정보 탐지 가능
- 머신러닝 기반 데이터 분류
- Finding 제공 (탐지 서비스)
- 대응은 별도 구성 필요 (Lambda 등)

---

### ⚖️ 서비스 비교

| 서비스 | 역할 |
|--------|------|
| WAF | L7 공격 차단 |
| Shield | DDoS 방어 |
| GuardDuty | 위협 탐지 (로그 기반) |
| Macie | S3 데이터 보안 / 민감 정보 탐지 |

---

### 💡 한 줄 정리
> Amazon Macie는 S3 데이터를 분석하여  
> 민감 정보(PII)와 데이터 노출 위험을 탐지하는 서비스이다.
